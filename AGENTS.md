# AGENTS.md

本文件是本项目后续 Agent 的项目级工作规则。目标是服务“鹭岛青年筑梦AI创新大赛·电商黑客松”，在现场开发约束下完成一个真实、可运行、可展示、可解释的 AI 电商最小闭环 Demo。

## 1. 启动检查

每次新任务开始前，按顺序读取：

1. `AGENTS.md`
2. `MEMORY.md`
3. `开发日志.md`
4. `README.md`
5. `docs/` 下与当前任务相关的文档

若当前目录不是 Git 仓库，必须明确记录该状态；正式开发前建议初始化 Git 仓库，用于证明现场开发过程和提交 GitHub 链接。

## 2. 比赛硬约束

- 主题：AI + 电商，从真实痛点到方案。
- 不做“大而全”的平台化设想，优先一个明确场景的最小闭环 Demo。
- Demo 必须可运行、可交互、可解释输入、处理和输出。
- 核心代码、核心逻辑、核心创意必须在比赛规定时间内现场完成。
- 允许使用公开开源库、公共 API、公开数据集、基础框架和通用工具。
- 禁止携带或伪装赛前完整项目，禁止抄袭，禁止跨队共享核心代码、逻辑、数据。
- 提交物：PPT、GitHub 链接、5 分钟以内录屏。
- 评分：技术实现 40 分，创意/商业价值 30 分，完成度 30 分。
- 项目真实性为一票否决项。

## 3. 推荐默认方向

本项目确定方向为"跨境电商开店注册引导工具"，聚焦亚马逊（Amazon）平台：

面向中国跨境电商卖家，通过 AI 引导完成亚马逊开店注册全流程（资质准备→资料填写→审核跟踪），降低新手卖家的注册门槛和驳回率。

核心 AI 能力：智能资质检查、资料填写引导、常见驳回原因预警、审核状态跟踪建议。

## 4. Agent 工作流

复杂任务必须按以下线性门禁执行：

1. Search Agent：调研外部资料、竞品、API、成本和可落地方案。
2. Explore Agent：只读检查当前代码库、文档、配置、历史记录。
3. Plan Agent：输出 SPEC、PLAN、TASKS 或可执行计划。
4. Worker Agent：按任务实现或写文档。
5. Verification Agent：运行构建、测试、检查或文档验收。
6. Evolution Agent：清理状态、更新记忆、沉淀经验。

主控 Agent 是 Coordinator，负责信息汇总、冲突裁决和最终质量把关。

## 5. SDD 规范

复杂开发遵循规格驱动开发：

```
模糊需求 -> SPEC.md -> PLAN.md -> TASKS.md -> 质量验证 -> 单任务实现 -> 测试 -> Review -> 下一个任务
```

必须先明确：

- 目标用户
- 真实痛点
- 输入、AI 处理、输出
- MVP 范围
- 非目标
- 验收标准
- 测试命令
- 演示路径

禁止让 AI 一次性生成完整工程。每个任务必须可独立执行、可验收、可回退。

## 6. Rules 与 Memory

- `AGENTS.md` 是团队规则，优先级高，用来约束所有 Agent 行为。
- `MEMORY.md` 是技术记忆，记录架构决策、当前状态、验证证据、遗留问题。
- `开发日志.md` 是统一开发日志，记录每次Session的进展、文件变更和下一步计划。

每次任务结束前，必须更新 `MEMORY.md` 和 `开发日志.md`。

## 7. 上下文管理

- 优先引用具体文件，而不是让 Agent 盲搜。
- 上下文超过约 60% 时拆分任务，避免一个 Agent 承担过大上下文。
- 子 Agent 只拿完成任务需要的材料。
- 对 PDF、接口文档、规则文档要先提炼再执行，避免把长文档直接塞给 Worker。

## 8. 安全和真实性

- 永远不要硬编码 API Key、Token、账号密码。
- `.env.local` 只保存本地密钥，不提交 Git。
- 合规功能只能写成“提示、缺口扫描、待补字段清单、草稿生成”，不能宣称自动完成法律合规。
- 使用 mock/demo data 时必须明确标注。
- 不要把计划中的功能写成已经实现。

## 9. 完成定义

一次任务完成必须满足：

- 改动文件明确。
- 验证步骤明确，最好有命令输出证据。
- 未完成项和风险明确。
- `MEMORY.md` 和 `开发日志.md` 已更新。

## 10. 中文标注规范

本项目所有代码产出必须遵循中文标注规则：

- **代码注释**：所有注释使用中文（包括函数说明、行内注释、TODO标记）
- **Git分支名**：使用中文拼音或中文描述（如 `feature/注册引导流程`）
- **Git提交信息**：使用中文描述变更内容
- **变量/函数命名**：保持英文（遵循JavaScript/TypeScript惯例），但JSDoc注释使用中文
- **文档**：所有项目文档使用中文
- **UI文案**：面向中国卖家用户，界面文案使用中文

## 11. 开发日志规范

每次开发Session结束后，必须更新 `开发日志.md`：

- **时间戳**：记录Session日期
- **解决的问题**：本次Session完成了什么
- **文件映射表**：本次新增/修改/删除了哪些文件
- **当前进度**：项目整体完成到哪一步
- **下一步**：下次Session应该从哪里开始
