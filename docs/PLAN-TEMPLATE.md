## 4. 实现顺序

推荐顺序：

1. 定义类型和 mock 数据。
2. 实现静态 UI 和结果展示。
3. 实现 API Route。
4. 接入 AI Prompt 和 JSON 解析。
5. 加入错误处理和 mock 降级。
6. 联调端到端流程。
7. 准备部署、录屏和 PPT。

## 5. 风险与降级

| 风险 | 影响 | 降级方案 |
|------|------|----------|
| AI API 不稳定 | 核心生成失败 | 使用 mock 结果完成演示 |
| 时间不足 | 功能不完整 | 只保留 P0 单页闭环 |
| 合规表述过度 | 误导用户/评委 | 只写缺口提示和人工确认 |
| 部署失败 | 无线上链接 | 本地演示 + 录屏兜底 |
| Git 未初始化 | 影响提交和真实性证明 | 尽早初始化并推送 GitHub |

## 6. 验证计划

### 6.1 本地验证
- [ ] `npm run dev` 能启动。
- [ ] 页面能打开。
- [ ] 表单能提交。
- [ ] API 返回合法 JSON。
- [ ] AI 失败时 mock 降级可用。

### 6.2 Demo 验证
- [ ] 5 分钟内能讲清楚痛点、输入、AI 处理、输出、商业价值。
- [ ] 录屏路径稳定。
- [ ] PPT 中不夸大功能或合规能力。

### 6.3 提交验证
- [ ] GitHub 仓库可访问。
- [ ] README 说明运行方式。
- [ ] 环境变量不泄露。
- [ ] PPT、录屏、链接齐全。

## 7. 任务拆分入口

完成本 PLAN 后，再生成 `docs/TASKS.md`。每个任务必须包含：

- 编号和名称。
- 优先级。
- 依赖。
- 涉及文件。
- 输入。
- 输出。
- 验收标准。
- 测试命令或验证方式。
